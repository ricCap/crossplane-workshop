# Welcome to the Crossplane workshop ⏱️ 5m

Over the next three hours you will install [Crossplane](https://crossplane.io), teach it how to manage Kubernetes resources, and use it to ship a small application. You'll walk away with a working mental model of Crossplane's core objects — Provider, XRD, Composition, Composite Resource — and a cluster you can keep experimenting in afterwards.

## How the workshop is structured

The sidebar on the left lists the modules in reading order, grouped by phase:

| Section | Module | Kind | ⏱️ |
|---|---|---|---|
| | Introduction | content | 5m |
| | Cheatsheet | reference | — |
| | Connect to your cluster | task | 10m |
| Crossplane 101 | Install Crossplane | task | 15m |
| Crossplane 101 | Providers & your first managed resource | task | 20m |
| Crossplane 101 | Define an Application | task | 40m |
| Crossplane 101 | Modify your Application | task | 20m |
| | Wrap-up | content | 5m |

The **task modules** (Connect to your cluster, plus everything in Crossplane 101) follow the same rhythm: a short framing, then hands-on steps, then a validation check. The Introduction, Cheatsheet, and Wrap-up are **content modules**: read and move on.

The **Cheatsheet** is a reference page. Keep it open in a second tab — it's the fastest way to look up what an XRD or a Composition is without leaving your current module.

If you're running this on your own laptop outside the live workshop, jump to [Solo local setup (k3d)](./solo-local-setup) first.

## How validation works

Each task module ends with a **check button**. Clicking it asks the workshop's validator to inspect your cluster and confirm the step worked. Green means you're good to move on; red comes with a diagnostic message so you know what to fix.

You're working in a pair. Enter your **pair ID** — the phrase your instructor handed you, like `fancy-lemon` — once, and every check on every module will use it:

Your pair ID is saved in your browser, so you can close this tab and come back. To switch, click **change** above.

### Try it now

The button below runs your first check. It asks the validator to list namespaces in your workshop cluster — a smoke test that confirms your pair ID matches a live cluster and that the workshop infrastructure can reach it.

If the tile turns green, you're ready for Connect to your cluster. If it turns red, re-check the pair ID your instructor gave you and ask them to verify your cluster is up before you move on.

## How to use the assistant

Each machine has [Claude Code](https://claude.com/claude-code) installed with the `crossplane-workshop-helper` plugin. When something is confusing or a check won't go green, type `/debug` and paste the error — the assistant will walk through what it sees. `/explain-xr` takes a pasted XR or Composition and explains it line by line.

The assistant doesn't know the answers ahead of time. It reads your cluster state, reads the official Crossplane docs, and tells you what it finds. If it guesses, call it out.
