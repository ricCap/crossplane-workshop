# Choose your own adventure — Panoramica

Un journey è uno scenario reale ricucito a partire dai task standalone sotto [2XX](../crossplane-2xx/overview), [3XX](../crossplane-3xx/overview) e [4XX](../crossplane-4xx/overview). La pagina inquadra lo scenario, suggerisce un ordine e linka ogni passo al modulo che fa al caso. I journey non sono task in sé — non c'è un check del validator, nessun golden path da copia-incollare. Scegli il journey che corrisponde a ciò che vuoi costruire, poi salti tra i moduli al tuo ritmo.

## Journey disponibili

| Journey | Scenario |
|---|---|
| [Template per microservizi](./microservice-templates) | Progetta una developer platform interna dove gli sviluppatori richiedono un `Application` opinionated e ottengono un golden path con database, repo GitHub e workload Kubernetes già cablati. |
| [Gestione dell'infrastruttura](./infrastructure-management) | Tira su una landing zone nel cloud per i team di sviluppo — VPC, subnet, security group — composta dietro un singolo XR per team. |
| [Automazione di processo](./process-automation) | Automatizza un processo multi-stage di lunga durata (security review, approvazioni multi-dipartimento) facendo polling su API esterne e avanzando attraverso gli stage fino al completamento. |

Sceglietene uno e tuffatevi. Potete sempre tornare indietro e provarne un altro.
