# Gestione dell'infrastruttura

> Gestisci il team di piattaforma. Un nuovo team di sviluppo viene onboardato e ha bisogno della propria landing zone nel cloud ā€” VPC, subnet, security group, le primitive di sempre. Vuoi un XR per team. Reviewabile in git, componibile e identico tra tutti i cloud che supporti.

## Cosa stai costruendo

Un XR `XLandingZone` (prefisso `X` per la [convenzione di naming](../cheatsheet#naming-convention-xr-kinds-start-with-x)) che prende un nome di team e una region. La Composition riconcilia le primitive di rete (VPC, subnet, security group) per quel team nel cloud di tua scelta, con default sensati cotti dentro. Aggiungere un nuovo team ĆØ un solo file YAML; cancellarne uno fa il rollback di tutto.

## Passi suggeriti

1. Leggi prima il [Pattern di setup di un Provider](../crossplane-2xx/provider-setup-pattern). Ogni provider in questo journey segue la stessa forma in tre passi ā€” install, ProviderConfig, MR ā€” e ne caberai almeno uno. `ClusterProviderConfig` vs `ProviderConfig` per-team ĆØ il punto decisionale che conta.
2. Scegli un cloud e percorri una MR bucket end to end: [provider-aws](../crossplane-3xx/provider-aws), [provider-gcp](../crossplane-3xx/provider-gcp) o [provider-azure](../crossplane-3xx/provider-azure). Il bucket ĆØ usa-e-getta ā€” le credenziali e il `ClusterProviderConfig` che metti su sono ciĆ² che riuserai.
3. Su Aruba si applica la stessa forma una volta che `provider-aruba` atterra in 3XX ā€” CRD Aruba VPC / subnet / security-group, cablati attraverso lo stesso flusso di install + `ClusterProviderConfig`.
4. Sostituisci il bucket con le primitive della landing zone. Sfoglia le CRD del provider sul [Crossplane Marketplace](https://marketplace.upbound.io/providers) ā€” ogni versione pubblicata elenca i kind che spedisce e i campi che accettano. VPC, subnet, security group, route table, IAM role ā€” scegli ciĆ² di cui la tua landing zone ha bisogno.
5. Avvolgi le primitive in un XRD + Composition. Riusa il pattern da [Definisci un'Application](../crossplane-101/define-application): un input XR (nome del team + region), diverse MR composte sotto.
6. Pacchettizza la landing zone come [Configuration](../crossplane-3xx/configurations) cosĆ¬ altri cluster ā€” staging, prod ā€” installano la stessa forma con un solo comando.

## Stretch goal
